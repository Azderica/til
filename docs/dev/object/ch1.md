## 2. 무엇이 문제인가

- 소프트웨어의 모듈은 세가지 목적이 있습니다.
  - 실행 중에 제대로 동작
  - 변경을 위해 존재
  - 코드를 읽는 사람과 의사소통

### 예상을 빗나가는 코드

- 이해 가능한 코드란 그 동작이 우리의 예상에서 크게 벗어나지 않는 코드입니다.
- 코드의 많은 세부사항은 모두에게 큰 부담을 줍니다.

### 변경에 취약한 코드

- 큰 문제는 변경에 취약합니다.
- 이는 객체 사이의 **의존성(dependency)** 과 관련된 문제입니다.
- 객체 사이의 의존성이 과한 경우, **결합도(coupling)** 가 높다고 합니다. 따라서 이를 낮추는 것이 필요합니다.
